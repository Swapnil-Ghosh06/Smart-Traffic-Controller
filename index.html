Project Technical Brief & Observations

An architectural deep-dive into SmartSignal's simulation engine and digital logic.

⚙️

Digital Logic Engine

The simulation's brain uses fundamental digital logic gates and sequential state machines to manage traffic flow.

  • SR Flip-Flops: Used as 1-bit memory cells to hold the current signal state. Set (S) and Reset (R) pulses switch the phase between North-South and East-West.
  • Priority Encoders: Resolve conflicts when multiple sensors fire. Emergency vehicles (A_x) carry the highest bit-priority, overriding standard density sensors (D_x).
  • Boolean Decision Logic: The controller evaluates complex algebraic expressions in real-time to decide if a phase transition is "Safe" or "Efficient."
🏎️

Kinematic Physics Engine

Unlike simple coordinate animations, SmartSignal uses a path-locked kinematic model for vehicle movement.

  • Path Centerlines: Vehicles follow strict 3D splines. Lateral offsets provide human-like "drift" within the lane without overstepping boundaries.
  • Kinematic Integration: Cars calculate acceleration and braking distance every frame using variable torque and friction constants.
  • Brake Zones: Signals inject virtual "stop blocks" into the lane paths. Vehicles react to these based on their current velocity and distance to the stop-line.
🧠

Key Observations

Based on continuous simulation testing, several critical efficiency insights have been discovered:

  • Throughput: Smart Mode typically achieves 22-35% higher vehicle clearance by eliminating "empty green" time (green lights with no cars).
  • Emergency Priority: By leveraging digital overrides, emergency vehicles experience zero wait time at the intersection, potentially saving critical seconds.
  • Environmental Impact: Reducing idling time directly impacts the CO2 and Fuel Wasted metrics. A 10-second reduction in avg wait scale up to kilograms of prevented emissions in high-density scenarios.
🧩

Component Significance

Induction Sensors (D_N, D_S...)

Simulate electromagnetic loops buried in the road. In the simulation, these detect vehicle proximity near the stop lines to estimate lane pressure.

Ambulance Transponders (A_N...)

Simulate V2I (Vehicle-to-Infrastructure) signals. When active, they bypass the standard timer and force an immediate phase change once safe.

Stop Lines & Signal Compliance

Strictly defined markers where the physics engine clamps vehicle movement. Stop lines are visible to help the user understand the "Compliance Zero-Point."

State Feedback Loop

The "Logic Circuit" visualization shows how data flows from sensors into logic gates. This bridges the gap between software simulation and physical hardware design.
